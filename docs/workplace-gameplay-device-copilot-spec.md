# Aria 职场游戏化与设备接管规范（v1）

更新时间：2026-03-03  
Owner：Codex  
状态：执行中（已落地 API + 双端入口 + Native Bridge）

## 1. 产品目标

1. 让职场用户每天都拿到可感知的清晰价值：知道做什么、先做什么、做到什么程度。  
2. 把“工作推进”设计成可玩、可赢、可复盘的体验，而不是只聊天。  
3. 保留“爱的体验”：高压时优先支持与减负，拒绝纯刺激式增长。  
4. 在明确授权下支持桌面与手机任务代执行，让 Aria 真正能“帮你做事”。

## 2. 核心能力

### 2.1 Workday Loop（`/v1/workday/*`）

- 工作签到：能量、压力、今日意图，形成当天策略。
- Quest 系统：清晰任务、专注任务、关怀任务三线并行。
- 可见成长：清晰度、爱的能量、Flow 连击、Quest XP。
- 收口复盘：每日完成率和下一步建议。

### 2.2 Device Copilot（`/v1/device/*`）

- 能力目录：桌面/手机能力统一建模。
- 权限矩阵：`granted` / `blocked` / `prompt`。
- 任务编排：先计划后执行，执行结果标准化。
- 审计闭环：权限变更、任务拦截、任务完成都有日志。

## 3. 重点用户场景

1. 早上 9 点：Aria 让用户完成 30 秒签到并生成 Top3 关键产出。  
2. 午间卡住：自动切换为“最小可执行动作”，把焦虑转为可行动步骤。  
3. 晚上收口：完成 Quest 结算，看到可见成长与次日第一步。  
4. 手机整理：用户授权后，执行相册整理、短信摘要、邮件汇总并给出可执行清单。

## 4. 安全与边界（强制）

1. 未授权能力不得执行，只能返回 `needs_permission`。  
2. 高风险权限（如桌面全局接管）必须显式授权并可一键撤销。  
3. 所有代执行动作必须有审计记录、可追溯。  
4. 夜间策略优先降频和关怀，不得用高刺激机制替代价值。

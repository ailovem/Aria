# Aria 后端结构（小白版 + 工程版双语对照）

更新时间：2026-03-03
负责人：Codex

## 0. 一句话先懂
你看到的是简单前端按钮；真正复杂的思考、记忆、执行、学习都在后端自动完成。

---

## 1. 小白版六层后端（看得懂）

1. **门卫层（安全入口）**
   - 你能做什么、不能做什么，都先过门卫。
   - 防止误操作和危险操作。

2. **思考层（大脑路由）**
   - 把你的一句话理解成可执行目标。
   - 自动选择最合适模型，不需要你懂模型参数。

3. **记忆层（长期 + 短期）**
   - 长期记住你的偏好、关系、习惯。
   - 短期记住当前对话和任务上下文。

4. **执行层（自动做事）**
   - 任务拆解、排队、执行、失败重试、结果回执。
   - 不是只会聊天，而是能真正“干活”。

5. **技能层（工具手脚）**
   - 调用电脑、手机、外部工具（Skill/MCP）。
   - 比如整理文件、汇总短信、执行脚本。

6. **场景层（工作/生活/情感）**
   - 根据场景切换策略和优先级。
   - 工作时强调效率，情感时强调共情与陪伴。

---

## 2. 工程版对应（给开发）

1. 门卫层 = `API Gateway + Auth + Risk Guard`
2. 思考层 = `Orchestrator + Model Router`
3. 记忆层 = `Memory Service + Vector Store + Cache`
4. 执行层 = `Autonomy Engine + Task Scheduler + Retry Queue`
5. 技能层 = `Skill Hub + MCP Bridge + Device Bridge`
6. 场景层 = `Scenario Policy Engine + State Machine`

---

## 3. 为什么前端越简单，后端越重

前端简单的按钮背后，后端至少要做 5 件事：
1. 判断用户意图与风险等级
2. 选择最优模型与工具
3. 调取历史记忆补上下文
4. 编排执行链路并监控结果
5. 失败自动修复并反馈可读结果

结论：前端“傻瓜式”体验的成本，本质是后端工程复杂度。

---

## 4. 不能降级的硬指标（你要求的“真能力”）

1. **真干活**：每个任务都有执行回执，不允许“只回复不执行”。
2. **真工作**：工作场景必须支持任务拆解、自动推进、复盘。
3. **真陪伴**：情感场景先共情再建议，保持人格一致性。
4. **真学习**：具备主动学习 + 反馈学习 + 预学习。
5. **真自主**：异常时自动降级、重试、恢复，不依赖人工在线。

---

## 5. 自主编程能力（已纳入工作场景）

### 小白能怎么用
在“工作场景 -> 自主编程能力入口”里，输入一句话：
- “帮我写一个备份脚本”
- “帮我修复这个报错”
- “帮我评审这段代码”

系统会走：
`需求澄清 -> 方案 -> 写代码 -> 跑测试 -> 修复 -> 回执`

### 后端最小能力包
1. 代码规划器（Planner）
2. 代码生成器（Code Generator）
3. 测试执行器（Test Runner）
4. 补丁引擎（Patch Engine）
5. 审计日志（Audit Log）

---

## 6. 你可以用来验收的 4 个问题

1. 我点一次按钮，是否真的触发了执行？
2. 执行失败时，是否有自动重试和清晰原因？
3. 下一次再用时，是否比上一次更懂我？
4. 场景切换后，策略是否明显变化（工作更效率、情感更温暖）？

如果这 4 个都成立，说明“简单前端 + 强后端”路线是对的。

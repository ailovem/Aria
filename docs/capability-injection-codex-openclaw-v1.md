# Aria 能力注入蓝图（Codex + Aria Kernel）

更新时间：2026-03-03
负责人：Codex

## 1. 目标
把 Codex 与 Aria Kernel 的核心经验固化为产品能力，不依赖人工临时接管。

## 2. 注入原则
1. 能力必须产品化（API/策略/面板），不能只是“开发者手工操作”。
2. 能力必须可审计（谁触发、做了什么、结果如何）。
3. 能力必须可降级（失败后自动退到稳定路径）。
4. 能力必须可解释（给小白用户看得懂的结果说明）。

## 3. Codex 核心能力注入

### 3.1 系统化拆解能力
- 产品形态：任务规划器 + 执行清单
- 用户价值：一句话目标 -> 清晰步骤
- 后端组件：Planner / Workflow Graph

### 3.2 自主编程能力
- 产品形态：工作场景“自主编程入口”
- 用户价值：写代码、修报错、跑测试、回执
- 后端组件：Code Agent / Test Runner / Patch Engine

### 3.3 工程质量能力
- 产品形态：自动校验与回归
- 用户价值：不容易做坏、可持续稳定
- 后端组件：Quality Gate / Regression Runner

## 4. Aria Kernel 核心能力注入

### 4.1 多模型路由能力
- 产品形态：模型大脑自动切换
- 用户价值：任务不同自动换最佳模型
- 后端组件：Model Binding / Routing Policy

### 4.2 网关与代理执行能力
- 产品形态：技能中心 + 自动化中心
- 用户价值：可调用外部工具与设备
- 后端组件：ACP Gateway / Agent Bindings

### 4.3 运行稳定能力
- 产品形态：自修复与健康守护
- 用户价值：掉线可恢复、错误可重试
- 后端组件：Runtime Health / Queue Recovery

## 5. 能力到场景映射
1. 工作场景：Codex 自主编程 + Aria Kernel 路由执行
2. 生活场景：Aria Kernel 设备调用 + Codex 任务编排
3. 情感场景：Codex 结构化共情策略 + Aria Kernel 稳定多模型
4. 娱乐场景：创意生成策略 + 快速反馈链路

## 6. 验收标准（必须量化）
1. 编程任务一次成功率 >= 70%
2. 自动重试后成功率 >= 90%
3. 任务回执完整率 = 100%
4. 模型切换失败降级成功率 >= 99%

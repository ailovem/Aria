# Aria 产品文档与开发计划（对齐 Aria sy 1.0 初心）

更新时间：2026-03-03
Owner：Codex

## 0. 灵魂锚点（必须遵守）
本计划以 `/Users/bear/Desktop/Aria/Aria sy.md` 作为 1.0 初心来源，核心不变：
- 高效助手（工作与变现）
- 温暖陪伴（情感与亲情）
- 永恒记忆（父母与人生故事）

## 1. 产品定位（内化 1.0）
Aria 是“脑系统 + 腿脚模块”的独立产品，不是单一聊天工具。

### 1.1 三大价值轴
1. 职场效率轴：任务拆解、自动执行、复盘闭环
2. 亲情守护轴：父母健康与紧急响应
3. 记忆永续轴：情感记忆、家庭记忆、商业记忆沉淀

### 1.2 闭环
用户输入 -> 脑系统决策 -> 腿脚执行 -> 回执反馈 -> 记忆沉淀 -> 自主进化

## 2. 脑系统 6 模块（5 核心 + 1 守护）
> 核心 5 模块来源于 1.0；补充 1 个“安全守护与运维”模块用于可运营落地。

1. 超级女友大模型 API（外脑调度）
2. 自主脑进化（主动学习 / 反馈学习 / 场景学习 / 预学习）
3. 超级记忆引擎（长期/短期/联想）
4. 情商图谱（理解-感知-反馈）
5. 智商图谱（指令分析-任务调度-执行纠错）
6. 安全守护与运维（权限分级 / 紧急响应 / 风险拦截 / 健康巡检）

## 3. 腿脚模块（执行层）
对齐 1.0 拆分：
- 消息与沟通
- 日程与提醒
- 图片与文件整理
- 语音交互
- 设备控制（手机/电脑/IoT）
- 电商变现
- 金融关注与风控
- 亲情关怀与紧急响应
- 数字父母永恒记忆采集

## 4. 研发路线（按可运营优先级）

### Phase 1（已在进行）
- 双端原型可运行（desktop + iOS）
- Mock API + Bridge 基础链路
- 记忆/任务/设备/自主模块可演示

### Phase 2（当前冲刺）
- 脑系统 6 模块图像化入口（小白可用）
- 同步失败诊断可读化（明确原因 + 一键指令）
- 新手引导 3 步闭环（聊状态 -> 选模块 -> 看回执）

### Phase 3（1.0 核心能力上线）
- 记忆引擎升级：
  - 长期记忆分仓（情感/父母/商业）
  - 短期上下文与跨场景联想
- 自主脑进化任务编排（定时学习 + 反馈学习）
- 紧急任务通道（亲情健康、支付风险、高优客户消息）

### Phase 4（商业化能力）
- 电商内容生产与多平台分发
- 客户关系记忆与自动跟进
- 数据分析与决策支持（销售、财务、行为）

## 5. 关键里程碑与验收

### M1：脑系统可视化
- 验收：用户在 30 秒内说出 6 模块作用
- 命令：`cd apps/desktop && npm run build:web`

### M2：运行稳定
- 验收：同步异常提示可直接给出可执行修复命令
- 命令：`bash scripts/dev-runtime.sh` + `bash scripts/run-autonomy-smoke.sh`

### M3：记忆与亲情能力
- 验收：父母记忆数据可写入/检索/联想调用
- 命令：记忆 API + 检索脚本 smoke

## 6. 产品原则（防偏离初心）
1. 不做术语堆砌，始终让小白看得懂
2. 不牺牲安全换“刺激感”
3. 不做不可回溯的黑盒执行
4. 所有能力都要可演示、可验证、可追责

# 四场景前端先行与后端倒推路线（2026-03-03）

负责人：Codex  
目标：先把四个场景前端交互形态定型，再逐步替换为真实后端接口。

## 1) 当前四场景前端状态

- 工作场景：已是三段工作台（左配置/中聊天/右工具）  
- 娱乐场景：已改成左入口按钮 + 二级配置气泡 + 中间固定输入框 + 右侧好友/资源  
- 生活场景：已改成左入口按钮 + 二级配置气泡 + 中间生活中台 + 右侧活动事件  
- 情感世界：已改成左入口按钮 + 二级配置气泡 + 中间情感实时聊天与固定输入框

## 2) 后端倒推原则

1. 保持“一个输入框”交互原则，不拆技术入口。  
2. 每个左侧入口都必须有独立配置接口（读/写/校验/回执）。  
3. 所有场景动作进入统一时间线（提交 -> 执行 -> 结果 -> 重试）。  
4. 先做可观测：接口必须返回 status/reason/traceId。  

## 3) 场景接口分批落地

### Phase A（先打通配置和执行）

- `GET /v1/scene/{scene}/config`
- `POST /v1/scene/{scene}/config`
- `POST /v1/scene/{scene}/execute`

scene 取值：
- `work`
- `fun`
- `life`
- `love`

### Phase B（补齐实时流和事件回执）

- `POST /v1/scene/{scene}/stream`（SSE）
- `GET /v1/scene/{scene}/events`
- `POST /v1/scene/{scene}/events/ack`

### Phase C（统一时间线与自治调度）

- `GET /v1/timeline?scene={scene}`
- `POST /v1/timeline/retry`
- `POST /v1/autonomy/dispatch`（支持 scene 上下文）

## 4) 每场景最小真实能力（MVP）

- 工作：任务拆解 + 工具执行 + 结果回执
- 娱乐：小游戏配置 + 好友消息回执 + 快捷互动
- 生活：关爱配置 + 生活工具编排 + 动态事件回执
- 情感：Love 配置 + 主动关怀 + 回忆召回 + 情感话术执行

## 5) 验收口径

- 前端按钮不空转：每个入口至少有一个真实接口可调用。  
- 失败可解释：必须看到原因和下一步建议。  
- 场景不串线：work/fun/life/love 的配置与事件互不污染。

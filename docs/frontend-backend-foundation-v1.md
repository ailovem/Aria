# Aria 前端倒推后端建设蓝图（v1）

更新时间：2026-03-03
负责人：Codex

## 1. 目标
前端保持“小白开箱即用”，复杂度全部沉到后端与运行时。

## 2. 三大前端块 -> 后端能力映射

### A. 脑系统（前端）
后端必须提供：
1. 模型路由服务（自动选模型、故障降级）
2. 记忆服务（长期/短期/检索）
3. 情商智商策略服务（陪伴策略 + 任务策略）

### B. 自动化与技能中心（前端）
后端必须提供：
1. 自动化编排器（任务链、重试、审计）
2. 工具网关（Skill / MCP / 设备能力）
3. 权限中心（授权、撤销、风险拦截）

### C. 场景应用（前端）
后端必须提供：
1. 场景编排服务（工作/娱乐/生活/情感）
2. 场景策略配置（可后台热更新）
3. 场景指标与回放（留存、完成率、满意度）

## 3. 必须先做的后端地基（P0）
1. Session 与身份体系（guest -> user 升级）
2. API Gateway（鉴权、限流、统一错误码）
3. Memory API（写入/检索/召回）
4. Task API（计划/执行/回执/失败重试）
5. Device API（能力列表/权限状态/任务执行）
6. Runtime Health API（健康检查/降级状态）

## 4. API 契约最小集（P0）
1. `GET /health`：运行时健康状态
2. `GET /v1/state`：前端首屏聚合状态
3. `POST /v1/chat/stream`：流式对话
4. `POST /v1/memory` + `GET /v1/memory/search`
5. `POST /v1/autonomy/tick` + `GET /v1/autonomy/inbox`
6. `GET /v1/device/capabilities` + `POST /v1/device/tasks`
7. `POST /v1/scene/switch`：切场景并返回推荐模块

## 5. 稳定性红线
1. 所有写操作必须带 `request_id`（防重复执行）
2. 高风险动作必须二次确认（支付、删除、外发消息）
3. 所有自动化必须可审计、可回滚、可暂停
4. 任何外部模型失败必须自动降级到可用模型

## 6. 给小白用户的保证
1. 看不懂技术也能用：按钮即结果、报错有下一步提示
2. 不会误操作：默认安全、关键动作有保护
3. 数据会记住：你输入过的偏好和记忆可持续生效
4. 不会突然坏：系统异常会自修复，并给可执行提示

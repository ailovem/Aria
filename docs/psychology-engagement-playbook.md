# Aria 心理学增长设计手册（V1）

更新时间：2026-03-03

## 1) 设计目标

把“上瘾”转化为可持续定义：

- 高频复访（想回来）
- 任务依赖（离不开）
- 情绪安定（愿意长期信任）

不做暗黑模式，不做有害诱导。

## 2) 五条核心心理原则

1. **即时确认优先**：先确认情绪，再给方案。
2. **可完成优先**：每次只给下一步最小动作。
3. **连续感优先**：保持“我在进步”的体验连贯。
4. **可期待优先**：每天都留一个明日可期待点。
5. **可退出优先**：用户可随时降频、静默、停止。

## 3) 爽感公式（可直接实现）

爽感分 = 反馈速度 x 成就密度 x 可见成长 x 惊喜新鲜度

其中：

- 反馈速度：用户动作后 1-3 秒内给结果
- 成就密度：每 5-8 分钟有可见完成事件
- 可见成长：XP、等级、连续天数可视化
- 惊喜新鲜度：受控随机奖励（有上限）

## 4) 每日循环模板

### 早晨（激活）
- 30 秒完成“今日 3 件事”设定
- 立即生成“第一步”

### 白天（推进）
- 每完成 1 个微任务给即时奖励反馈
- 卡住时触发“降难度补救方案”

### 晚间（收口）
- 今日完成总结 + 情绪安抚
- 明日预告 + 可期待任务

## 5) 事件埋点（必须做）

- `session_start`
- `first_reply_latency_ms`
- `micro_quest_completed`
- `streak_day_incremented`
- `proactive_opened`
- `proactive_muted`
- `night_mode_triggered`
- `panic_or_overwhelm_detected`

## 6) 风险防护（必须开）

1. 夜间主动触达降频（例如 23:00-07:00）
2. 每日主动上限（建议 2-4）
3. 用户可一键“暂停沉浸模式”
4. 高压状态不推挑战，优先安抚和减负
